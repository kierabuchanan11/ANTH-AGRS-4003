# ANTH-AGRS-4003
This project analyses excavation reports from Pompeii’s Regio II to investigate whether AI-inspired text analysis techniques, such as computational text analysis, can distinguish between domestic and religious spaces based solely on the language used in archaeological documentation. By looking at the descriptive terminology archaeologists use, the project examines whether patterns in word choice align with the functional categorization of sites and whether these patterns can be used to classify previously unlabelled reports. Table 1 lists the sites included in this study, providing a clear reference for the domestic and religious spaces analysed. Excavation reports are a particularly rich resource because they combine detailed observations of architectural features, rooms, and objects with interpretive commentary, offering insight into how archaeologists describe and conceptualize different types of spaces (Pompeii Sites, 2016).

The digital component of the project uses Voyant Tools to conduct text analysis, including word frequency counts, keyword extraction, and visualizations. I begin by uploading a corpus of reports that have already been classified as domestic or religious using information from the Pompeii Sites website. Voyant Tools processes the texts and generates word frequency lists, trend graphs, and word clouds, which help identify the terms most commonly associated with each category. For example, domestic reports often include words like “house,” “room,” and “courtyard,” while religious reports frequently use terms such as “altar,” “temple,” and “god.” Examining these frequency patterns allows me to create keyword lists that capture the distinctive language used to describe each type of space (Richards, Tudhope, & Vlachidis, 2015) (see Figures 1 & 2).

It is important to note that some words identified by Voyant Tools were removed from the keyword list because they were too broad or common. Including such generic terms risked misclassifying sites, so words like “century” or “eruption”—which do not directly indicate the type of site—were excluded from the analysis (see the full script to see what keywords were used).
Once the keyword lists are compiled, they are applied in a Python code-based classification model. This model evaluates unlabelled reports by counting the number of domestic and religious keywords and assigning a category based on whichever appears more frequently. Figure 1 shows the Python code used for this process. For this project, I used every excavation report listed under “Regio VII” in the Pompeii Sites’ book A Guide to the Pompeii Excavations (2016), carefully inputting each report into my Python classification code. This allowed me to systematically test the model on the entire set of site descriptions, recording how often the code correctly identified reports as domestic or religious based on the keyword lists generated through Voyant Tools. By doing this, the project not only evaluates the potential of AI-inspired text analysis for site classification but also highlights patterns of accuracy and misclassification across a complete, site-specific dataset.
The line of code if domestic_count < 2 and religious_count < 2: return "Unclear" was included to ensure the script would not misclassify sentences like: “On the sides of the Temple of Jupiter, in the northern part of the Forum, there were two old honorary arches (pp. 88)” as religious solely because the phrase “Temple of Jupiter” appears. However, because this report also contains other words that align with the religious category, it was ultimately categorized as religious (Board of Cultural Heritage of Pompeii, 2016). Notably, the code’s main issue occurred with sites that were neither clearly domestic nor clearly religious, as it tended to classify these ambiguous spaces as religious. Overall, the script correctly identified 15 out of 21 reports, or approximately 71% (see Table1)
This outcome highlights a broader point about the limitations of automated classification: the challenges are not solely due to the script itself, but also reflect the variability and interpretive nature of archaeological reporting. Excavation reports often use overlapping terminology, author-specific conventions, or context-dependent descriptions that can make distinctions between domestic and religious spaces ambiguous. As a result, even a carefully designed keyword-based script can misclassify sites when the language is not clearly indicative of a single function. This suggests that while computational tools can assist in analysing large corpora of reports, the clarity and consistency of human-written documentation remain crucial. Academics should therefore be mindful that the language they choose in reporting can directly influence the reliability of automated or AI-inspired analyses, and standardized descriptive practices could enhance the effectiveness of such digital methods.
The analysis is further supported by primary archaeological sources, including detailed guides from the Board of Cultural Heritage of Pompeii (2016) and interactive maps from the Pompeii Sites project, which provide site-specific context and architectural details that help ground the analysis in accurate evidence. Integrating textual descriptions with spatial layouts allows the project to account for structural and contextual factors that may influence terminology. Richards, Tudhope, and Vlachidis (2015) also emphasize how text mining can extract useful information from unstructured archaeological documents, while acknowledging challenges like inconsistent terminology and variation across reports.
By combining Voyant-based text analysis, Python classification, and primary archaeological sources, this project contributes to both digital archaeology research and public understanding of how language shapes interpretation, while critically examining the strengths and limitations of automated approaches (Whittaker, Caulkins, & Kamp, 1998; Huggett, 2013; Bellat et al., 2025). Academically, it shows how digital tools can complement traditional archaeological methods by systematically analysing large amounts of text, revealing patterns in terminology, and offering insight into how archaeologists conceptualize and communicate their findings. Methodologically, it highlights how computational analysis can serve as a preliminary categorization tool, quickly identifying recurring words or anomalies, while also emphasizing that automated classification alone cannot capture nuanced context, interpretive choices, or author biases. Ultimately, this project demonstrates the value of combining computational methods, careful interpretation, and primary evidence, showing how AI-inspired text analysis can enhance archaeological research while making clear that meaningful conclusions still rely on human expertise. It also provides a model for the responsible use of digital tools in studying and interpreting the past.
